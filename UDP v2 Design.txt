UDP v2 Design

* [Done] 加密
	加密默认对UDP整包做块加密。
	因为小状态包的存在，对Data单独的流加密强化是必要的
	一个 ecdh，通过 uint16_t 的 IV mask 处理 secret 得到第二个 secret。
	加密深度：选择整个package body，或者仅 Data 部分
	可丢弃包深度加密采用块加密，非可丢弃包深度加密采用流加密

	** 因为深度加密影响性能和提前拆解包，所以暂时保留，不实现 **

* 可丢弃包的指纹验证
	可丢弃包也需要验证指纹
	可丢弃包序号依旧随机。
	当第一个可靠包到达的3秒（可毫秒级配置）后，所有收到的可丢弃包君需要验证。

* 优化

	NX 的带宽消耗高于洲际。怀疑是因响应延迟在微秒级别，导致 流控 internal 为 0，然后立刻频繁重发导致。

收尾注意事项：

* 目前提前拆解包没有开放配置接口。默认开启。只有启用了数据强化加密才会被关闭。
  但数据强化加密会影响效率和性能。

* 目前没有提供 answer 的重发过期时间配置。默认为：
	Server 正常发送：UDPServer 的 timeout 配置时间
	Server 提前返回：UDPServer 的 timeout 配置时间
	Client 正常发送：UDPClient 的 timeout 配置时间
	Client 提前返回：ClientEngine 的 timeout 配置时间
